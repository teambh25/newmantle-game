# Game V2 API

Base URL: `/v2`

인증은 **선택사항**입니다. `Authorization: Bearer <token>` 헤더가 있으면 유저를 식별하여 통계를 기록하고, 없어도 게임 플레이는 정상 동작합니다.

---

## 1. GET `/v2/quizzes/{date}/guess/{word}` - 단어 추측

입력한 단어의 유사도 점수와 순위를 반환합니다. 정답일 경우 정답 정보를 함께 반환합니다.
로그인 상태라면 추측 기록이 통계에 반영됩니다.

### 요청

| 파라미터 | 타입     | 위치 | 필수 | 설명                                     |
|---------|----------|------|------|------------------------------------------|
| `date`   | `date`   | path | O    | 퀴즈 날짜 (ISO 8601, 예: `2026-03-16`)     |
| `word`   | `string` | path | O    | 추측할 단어 (한글)                          |

```
GET /v2/quizzes/2026-03-16/guess/사과
Authorization: Bearer <token>  (선택)
```

### 응답 `200 OK`

**오답인 경우**

```json
{
  "correct": false,
  "rank": 152,
  "score": 45.23,
  "answer": null
}
```

**정답인 경우**

```json
{
  "correct": true,
  "rank": null,
  "score": null,
  "answer": {
    "word": "사과",
    "tag": "명사",
    "description": "사과나무의 열매. 둥글고 빨갛게 익으며 새콤달콤한 맛이 난다."
  }
}
```

### 응답 스키마

**`GuessResp`**

| 필드      | 타입             | 설명                                    |
|----------|------------------|-----------------------------------------|
| `correct` | `bool`           | 정답 여부                                |
| `rank`    | `int \| null`    | 유사도 순위 (정답이면 `null`)               |
| `score`   | `float \| null`  | 유사도 점수 (정답이면 `null`)               |
| `answer`  | `Answer \| null` | 정답 정보 (오답이면 `null`)                |

**`Answer`**

| 필드          | 타입     | 설명        |
|--------------|----------|-------------|
| `word`        | `string` | 정답 단어    |
| `tag`         | `string` | 품사 태그    |
| `description` | `string` | 단어 설명    |

### 에러

| 상태 코드 | 발생 조건                                    | 응답 예시                                    |
|----------|---------------------------------------------|----------------------------------------------|
| `404`    | 추측한 단어가 퀴즈 데이터에 존재하지 않을 때        | `{"detail": "Invalid guess request"}`        |
| `500`    | 정답 데이터를 서버에서 찾을 수 없을 때 (서버 오류)   | `{"detail": "Can't find answer"}`            |
| `422`    | path 파라미터 형식이 잘못되었을 때                 | FastAPI 기본 validation error 형식            |

## 2. GET `/v2/quizzes/{date}/hint/{rank}` - 힌트 조회

특정 순위의 단어를 힌트로 반환합니다. `rank=0`일 경우 정답의 초성을 반환합니다.
로그인 상태라면 힌트 사용 기록이 통계에 반영됩니다.

### 요청

| 파라미터 | 타입   | 위치 | 필수 | 제약 조건                  | 설명             |
|---------|--------|------|------|--------------------------|------------------|
| `date`   | `date` | path | O    | -                        | 퀴즈 날짜          |
| `rank`   | `int`  | path | O    | `0 <= rank <= max_rank`  | 조회할 순위        |

```
GET /v2/quizzes/2026-03-16/hint/1
Authorization: Bearer <token>  (선택)
```

### 응답 `200 OK`

**rank >= 1인 경우 (단어 힌트)**

```json
{
  "hint": "바나나",
  "score": 48.71
}
```

**rank = 0인 경우 (정답 초성 힌트)**

```json
{
  "hint": "ㅅㄱ",
  "score": null
}
```

### 응답 스키마

**`HintResp`**

| 필드    | 타입             | 설명                                          |
|--------|-----------------|-----------------------------------------------|
| `hint`  | `string`        | rank >= 1: 해당 순위의 단어 / rank = 0: 정답 초성 |
| `score` | `float \| null` | 유사도 점수 (rank = 0이면 `null`)                |

### 에러

| 상태 코드 | 발생 조건                                    | 응답 예시                                    |
|----------|---------------------------------------------|----------------------------------------------|
| `404`    | 해당 순위에 데이터가 존재하지 않을 때              | `{"detail": "Invalid hint request"}`         |
| `422`    | `rank`가 범위를 벗어나거나 형식이 잘못되었을 때     | FastAPI 기본 validation error 형식            |

## 3. GET `/v2/quizzes/{date}/give-up` - 포기

정답 정보를 반환합니다. 미래 날짜에 대해서는 포기할 수 없습니다.
로그인 상태라면 포기 기록이 통계에 반영됩니다.

### 요청

| 파라미터 | 타입   | 위치 | 필수 | 설명                                     |
|---------|--------|------|------|------------------------------------------|
| `date`   | `date` | path | O    | 퀴즈 날짜 (ISO 8601, 예: `2026-03-16`)     |

```
GET /v2/quizzes/2026-03-16/give-up
Authorization: Bearer <token>  (선택)
```

### 응답 `200 OK`

```json
{
  "word": "사과",
  "tag": "명사",
  "description": "사과나무의 열매. 둥글고 빨갛게 익으며 새콤달콤한 맛이 난다."
}
```

### 응답 스키마

**`GiveUpResp`** (`Answer`와 동일)

| 필드          | 타입     | 설명        |
|--------------|----------|-------------|
| `word`        | `string` | 정답 단어    |
| `tag`         | `string` | 품사 태그    |
| `description` | `string` | 단어 설명    |

### 에러

| 상태 코드 | 발생 조건                                    | 응답 예시                                    |
|----------|---------------------------------------------|----------------------------------------------|
| `422`    | 미래 날짜를 요청했을 때                          | `{"detail": "Invalid give up request"}`      |
| `404`    | 해당 날짜의 정답 데이터가 존재하지 않을 때          | `{"detail": "Invalid give up request"}`      |
| `422`    | path 파라미터 형식이 잘못되었을 때                 | FastAPI 기본 validation error 형식            |
