# Game V2 API

Base URL: `/v2`

인증은 **필수**입니다. 아래 중 하나의 헤더를 포함해야 합니다. 둘 다 없거나 둘 다 있으면 `401`을 반환합니다.

| 헤더 | 설명 |
|------|------|
| `Authorization: Bearer <token>` | 회원 (Supabase JWT) |
| `X-Guest-Id: <uuid>` | 게스트 (클라이언트 생성 UUID v4) |

## 접근 가능 기간

모든 엔드포인트는 **오늘과 어제**(최근 2일, KST 기준)의 퀴즈에만 접근할 수 있습니다. 그 외의 날짜(미래 또는 2일 이전)를 요청하면 `422`를 반환합니다.

---

## 1. GET `/v2/quizzes/{date}/guess/{word}` - 단어 추측

입력한 단어의 유사도 점수와 순위를 반환합니다. 정답일 경우 정답 정보를 함께 반환합니다.
추측 기록은 통계에 반영됩니다.

### 요청

| 파라미터 | 타입     | 위치 | 필수 | 설명                                     |
|---------|----------|------|------|------------------------------------------|
| `date`   | `date`   | path | O    | 퀴즈 날짜 (ISO 8601, 예: `2026-03-16`)     |
| `word`   | `string` | path | O    | 추측할 단어 (한글)                          |

```
GET /v2/quizzes/2026-03-16/guess/사과
Authorization: Bearer <token>
```

또는

```
GET /v2/quizzes/2026-03-16/guess/사과
X-Guest-Id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

### 응답 `200 OK`

**오답인 경우**

```json
{
  "correct": false,
  "rank": 152,
  "score": 45.23,
  "answer": null
}
```

**정답인 경우**

```json
{
  "correct": true,
  "rank": null,
  "score": null,
  "answer": {
    "word": "사과",
    "tag": "명사",
    "description": "사과나무의 열매. 둥글고 빨갛게 익으며 새콤달콤한 맛이 난다."
  }
}
```

### 응답 스키마

**`GuessResp`**

| 필드      | 타입             | 설명                                    |
|----------|------------------|-----------------------------------------|
| `correct` | `bool`           | 정답 여부                                |
| `rank`    | `int \| null`    | 유사도 순위 (정답이면 `null`)               |
| `score`   | `float \| null`  | 유사도 점수 (정답이면 `null`)               |
| `answer`  | `Answer \| null` | 정답 정보 (오답이면 `null`)                |

**`Answer`**

| 필드          | 타입     | 설명        |
|--------------|----------|-------------|
| `word`        | `string` | 정답 단어    |
| `tag`         | `string` | 품사 태그    |
| `description` | `string` | 단어 설명    |

### 에러

| 상태 코드 | 발생 조건                                    | 응답 예시                                    |
|----------|---------------------------------------------|----------------------------------------------|
| `401`    | 인증 헤더가 없거나 유효하지 않을 때                 | `{"detail": "Authentication failed"}`        |
| `422`    | 접근 불가 날짜(미래 또는 2일 이전)를 요청했을 때     | `{"detail": "Invalid guess request"}`        |
| `404`    | 추측한 단어가 퀴즈 데이터에 존재하지 않을 때        | `{"detail": "Invalid guess request"}`        |
| `500`    | 서버 내부 오류 (퀴즈 데이터 무결성 문제 등)        | `{"detail": "Server error"}`                 |
| `422`    | path 파라미터 형식이 잘못되었을 때                 | FastAPI 기본 validation error 형식            |

## 2. GET `/v2/quizzes/{date}/hint/{rank}` - 힌트 조회

특정 순위의 단어를 힌트로 반환합니다. `rank=0`일 경우 정답의 초성을 반환합니다.
힌트 사용 기록은 통계에 반영됩니다.

### 요청

| 파라미터 | 타입   | 위치 | 필수 | 제약 조건                  | 설명             |
|---------|--------|------|------|--------------------------|------------------|
| `date`   | `date` | path | O    | -                        | 퀴즈 날짜          |
| `rank`   | `int`  | path | O    | `0 <= rank <= max_rank`  | 조회할 순위        |

```
GET /v2/quizzes/2026-03-16/hint/1
Authorization: Bearer <token>
```

또는

```
GET /v2/quizzes/2026-03-16/hint/1
X-Guest-Id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

### 응답 `200 OK`

**rank >= 1인 경우 (단어 힌트)**

```json
{
  "hint": "바나나",
  "score": 48.71
}
```

**rank = 0인 경우 (정답 초성 힌트)**

```json
{
  "hint": "ㅅㄱ",
  "score": null
}
```

### 응답 스키마

**`HintResp`**

| 필드    | 타입             | 설명                                          |
|--------|-----------------|-----------------------------------------------|
| `hint`  | `string`        | rank >= 1: 해당 순위의 단어 / rank = 0: 정답 초성 |
| `score` | `float \| null` | 유사도 점수 (rank = 0이면 `null`)                |

### 에러

| 상태 코드 | 발생 조건                                    | 응답 예시                                    |
|----------|---------------------------------------------|----------------------------------------------|
| `401`    | 인증 헤더가 없거나 유효하지 않을 때                 | `{"detail": "Authentication failed"}`        |
| `422`    | 접근 불가 날짜(미래 또는 2일 이전)를 요청했을 때     | `{"detail": "Invalid hint request"}`         |
| `500`    | 서버 내부 오류 (해당 순위의 데이터 누락 등)         | `{"detail": "Server error"}`                 |
| `422`    | `rank`가 범위를 벗어나거나 형식이 잘못되었을 때     | FastAPI 기본 validation error 형식            |

## 3. GET `/v2/quizzes/{date}/give-up` - 포기

정답 정보를 반환합니다. 접근 가능 기간을 벗어난 날짜(미래 또는 2일 이전)는 포기할 수 없습니다.
포기 기록은 통계에 반영됩니다.

### 요청

| 파라미터 | 타입   | 위치 | 필수 | 설명                                     |
|---------|--------|------|------|------------------------------------------|
| `date`   | `date` | path | O    | 퀴즈 날짜 (ISO 8601, 예: `2026-03-16`)     |

```
GET /v2/quizzes/2026-03-16/give-up
Authorization: Bearer <token>
```

또는

```
GET /v2/quizzes/2026-03-16/give-up
X-Guest-Id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

### 응답 `200 OK`

```json
{
  "word": "사과",
  "tag": "명사",
  "description": "사과나무의 열매. 둥글고 빨갛게 익으며 새콤달콤한 맛이 난다."
}
```

### 응답 스키마

**`GiveUpResp`** (`Answer`와 동일)

| 필드          | 타입     | 설명        |
|--------------|----------|-------------|
| `word`        | `string` | 정답 단어    |
| `tag`         | `string` | 품사 태그    |
| `description` | `string` | 단어 설명    |

### 에러

| 상태 코드 | 발생 조건                                    | 응답 예시                                    |
|----------|---------------------------------------------|----------------------------------------------|
| `401`    | 인증 헤더가 없거나 유효하지 않을 때                 | `{"detail": "Authentication failed"}`        |
| `422`    | 접근 불가 날짜(미래 또는 2일 이전)를 요청했을 때     | `{"detail": "Invalid give up request"}`      |
| `500`    | 서버 내부 오류 (정답 데이터 누락 등)               | `{"detail": "Server error"}`                 |
| `422`    | path 파라미터 형식이 잘못되었을 때                 | FastAPI 기본 validation error 형식            |
